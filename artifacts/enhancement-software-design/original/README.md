# Original Artifact – Contact Service (CS-320)

## Overview
This folder contains the original version of the Contact Service application developed during CS-320. The application is designed to store and manage basic contact information.

## Functionality
The original application supports the following features:
- Creating new contact records
- Storing contact details (ID, first name, last name, phone number, address)
- Updating existing contact fields
- Deleting contact records

## Design Characteristics
- Basic object-oriented design using Java classes
- A service class responsible for managing contact data
- Unit tests included to verify core functionality

## Limitations (Pre-Enhancement)
The original version has several limitations:
- Limited input validation for user data
- No protection against duplicate contact IDs
- Minimal error handling and exception messaging
- Less structured separation of validation and business logic
- Limited testing coverage for edge cases and invalid inputs

## Purpose in CS-499
This original artifact is included as a baseline for comparison. It demonstrates the starting point before enhancements were applied. The enhanced version improves upon this design by introducing stronger validation, improved error handling, better data protection, and more comprehensive testing.
